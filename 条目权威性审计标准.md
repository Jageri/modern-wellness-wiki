# 权威性审计规范

**版本**: 2.0
**创建日期**: 2026-05-11
**更新日期**: 2026-05-12
**核心原则**: 绝对准确、绝不编造、承认不确定性

---

## ⛔ 三条红线（任何一条不通过即为不合格）

### 红线 1：引用必须真实存在
- 每条引用的文献必须实际存在，可通过 PubMed/Google Scholar/出版社网站验证
- 作者名、期刊名、年份、PMID 必须与原文完全一致
- **禁止编造引用**——即使结论正确，引用不存在即为不合格

### 红线 2：条目内容必须完全来自引用文献
- 条目中每个数据点（效果量、样本量、HR/OR/RR、置信区间）必须能在引用文献的原文中找到
- **禁止将 A 论文的数据归于 B 论文**
- **禁止编造数据**——即使方向正确，数字与原文不符即为不合格
- 验证方法：web_search 搜索原文 → web_fetch 获取摘要/全文 → 逐字核对

### 红线 3：必须有参考文献
- 每个条目必须至少引用 1 篇可验证的参考文献
- **禁止无参考文献的条目**——即使内容方向正确，无引用即为不合格
- 参考文献格式：作者 et al. 期刊. 年份. [URL](URL)

## 一、审计目标

对长寿百科的每个条目进行系统性权威性审计，确保：
1. 每个结论都有可靠来源支撑
2. 研究的局限性被充分披露
3. 不确定性被明确标注（"没有确定结论"本身是一个准确的结论）
4. 最新进展被及时纳入

## 二、审计维度（10 项）

每个条目必须通过以下全部检查。任何一项不通过，条目需要修正或标注。

### 1. 来源质量

**检查内容：**
- 引用是否来自同行评审期刊？
- 优先级：大型 RCT > Meta 分析/系统综述 > 前瞻性队列研究 > 横断面研究 > 病例报告 > 专家意见
- 是否引用了权威机构的立场声明（WHO、CDC、Lancet 编辑部等）？
- 排除预印本（未经同行评审）、掠夺性期刊、博客、自媒体

**不通过处理：** 替换为更可靠的来源，或标注"证据质量有限"

### 2. 结论一致性

**检查内容：**
- 用条目核心关键词搜索 PubMed/Cochrane/Google Scholar
- 主流证据是否支持条目结论？
- 条目是否过度简化了复杂结论？
- 数字是否与原文一致（不能"大约"、"差不多"）？

**不通过处理：** 修正结论，或标注"证据存在矛盾"

### 3. 反面证据

**检查内容：**
- 是否存在同等质量的研究得出相反结论？
- 条目是否只引用了支持方的研究而忽略了反对方？
- Cochrane 系统综述是否支持该结论？

**不通过处理：** 补充反面证据，将条目标注为"存在争议"，并说明争议双方的核心论据

### 4. 效果量准确性

**检查内容：**
- 引用的数字是否准确？与原文逐一核对
- 区分相对风险（RR/HR）与绝对风险（AR）
  - 相对风险降低 50% ≠ 绝对风险降低 50%
  - 例：基线风险 2%，干预后 1% → RR -50%，AR 仅 -1%
- 置信区间（CI）是否包含 1？（包含 1 = 不显著）
- NNT（需治疗人数）是否被忽略？

**不通过处理：** 修正数字，必要时同时标注相对风险和绝对风险

### 5. 利益冲突

**检查内容：**
- 研究是否由利益相关方资助？
  - 乳制品协会资助的牛奶研究
  - 巧克力厂商资助的可可研究
  - 补剂公司资助的维生素研究
- 作者是否与相关产业有财务关联？
- 独立研究是否得出相同结论？

**不通过处理：** 降权处理——在条目中注明"该研究由 [利益方] 资助"，并补充独立研究

### 6. 因果 vs 相关

**检查内容：**
- 条目是否把相关性说成了因果性？
- 观察性研究的结论是否被表述为因果？
- 是否有 RCT 支持因果推断？
- 混杂因素是否被充分控制？

**不通过处理：** 修正措辞
- ❌ "吃坚果降低死亡率"
- ✅ "吃坚果与死亡率降低相关（观察性研究，因果关系未确立）"

### 7. 人群适用性（重点）

**检查内容：**
- 研究样本的人群特征：
  - 年龄范围（65 岁以上？20-40 岁？）
  - 性别（仅男性？仅女性？混合？）
  - 种族/民族（亚洲人？白人？黑人？）
  - 健康状态（健康人群？慢性病患者？）
  - 地区（高收入国家？全球？）
- 结论是否可外推到其他人群？
- 是否存在已知的亚组差异？

**不通过处理：** 在条目中明确标注适用人群和不适用人群
- 例："本研究样本为 65 岁以上白人男性，结论对年轻女性的适用性尚不确定"

### 8. 研究局限性

**检查内容：**
- 研究本身的局限性是否被原始论文声明？
- 常见局限性：
  - 样本量不足
  - 随访时间过短
  - 选择偏倚（志愿者偏差、健康使用者偏差）
  - 测量误差（自报数据 vs 客观测量）
  - 失访率过高
  - 单中心研究
- 条目是否充分披露了这些局限性？

**不通过处理：** 在条目"证据"部分专门标注局限性

### 9. 发表偏倚

**检查内容：**
- 阳性结果是否更容易被发表？
- 是否存在漏斗图不对称？
- 是否有注册的临床试验未发表结果（可在 ClinicalTrials.gov 查询）？
- Meta 分析是否评估了发表偏倚？

**不通过处理：** 标注"可能存在发表偏倚"，效果量可能被高估

### 10. 时效性

**检查内容：**
- 主要引用是否来自近 5 年的研究？
- 是否有更新的 Meta 分析或 RCT 可以替代旧证据？
- 原始结论是否已被后续研究推翻或修正？
- 条目是否遗漏了近 1-2 年的重要进展？

**不通过处理：** 更新引用和结论，标注旧证据已被更新

## 三、审计流程

### 单条目审计流程

```
1. 提取条目核心结论和引用
2. 用关键词搜索 PubMed/Cochrane/Google Scholar
3. 筛选 Top 10 相关文献（按引用量/影响力排序）
4. 逐项对照 10 个审计维度
5. 输出审计报告：
   - 通过 / 需修正 / 需标注局限性
   - 具体修正建议
   - 缺失的反面证据
   - 最新进展
6. 根据审计报告修正条目
7. 更新条目的审计记录
```

### 审计报告模板

```markdown
## 审计记录

### 审计日期: YYYY-MM-DD
### 审计人: [名称]
### 审计结果: 通过 / 需修正 / 需标注

### 检查项:
- [ ] 来源质量
- [ ] 结论一致性
- [ ] 反面证据
- [ ] 效果量准确性
- [ ] 利益冲突
- [ ] 因果 vs 相关
- [ ] 人群适用性
- [ ] 研究局限性
- [ ] 发表偏倚
- [ ] 时效性

### 发现问题:
1. [问题描述]
2. [问题描述]

### 修正建议:
1. [修正内容]
2. [修正内容]

### 参考文献:
- [新增/更新的文献]
```

## 四、审计优先级

| 优先级 | 标准 | 目标 |
|--------|------|------|
| P0 | 效果量大（±30%+）、受众广 | 立即审计 |
| P1 | 有争议、来源质量存疑 | 优先审计 |
| P2 | 数据明确、来源可靠 | 常规审计 |
| P3 | 标注为"~"（不确定）的条目 | 补充最新证据 |

---

## 五、条目中的不确定性标注规范

研究结论不同时，按以下规范标注：

| 情况 | 标注方式 |
|------|---------|
| 多项 RCT 一致支持 | ⭐⭐⭐⭐⭐（因果关系确凿） |
| 大型队列研究一致 | ⭐⭐⭐⭐（强相关，因果待确认） |
| 有限研究支持 | ⭐⭐⭐（初步证据） |
| 研究结论矛盾 | ⭐⭐（存在争议），列出双方论据 |
| 仅动物/体外实验 | ⭐（人体证据不足） |
| 无直接研究 | 标注"无人体直接证据" |

---

## 六、审计频率

- **新增条目**: 发布前必须完成审计
- **已有条目**: 每季度轮审（按优先级排序）
- **重大事件**: 相关领域出现重磅研究时即时审计
- **年度审计**: 每年对所有条目做一次全面时效性检查

---

## 七、审计记录存储

- 每个条目的审计记录附加在条目文件末尾
- 审计日志汇总在 `audit-log.md`
- 审计统计在 `gap-tracker.md` 中维护
